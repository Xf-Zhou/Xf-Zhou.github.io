集合

什么是集合？

简称集，是用来存储多个元素的容器

集合和数组的区别

  • 元素类型

    ​ 集合：引用类型（存储基本类型时自动装箱）

    ​ 数组：基本类型、引用类型

  • 元素个数

     集合：不固定，可任意扩容

​ 数组：固定，不能改变容量

  • 集合的好处

    ​ 不受容器大小限制，可以随时添加、删除元素

    ​ 提供了大量操作元素的方法（判断、获取等）

Java的集合体系

  • 单列集合（Collection）

    ​ List：ArrayList

    ​ Set：HashSet

  • 双列集合（Map：key，value）

    ​ Map : HashMap

List集合的特点

  1. 特点：可重复、有序（存取顺序相同）
  2. 应用：List list = new ArrayList()

需求：向List集合中添加三个元素，并遍历打印

分析：

  • 向集合中添加元素的方法为：add()
  • 遍历集合的方式：for循环
  • 获取集合中元素个数的方法：size()

步骤：

  1. 创建集合对象：List list = new ArrayList()；

  2. 分别创建三个Student对象;

  3. 使用add方法将Student对象添加到集合中;

    list.add(s1);
list.add(s2);
list.add(s3);

  4. 使用for循环遍历集合并打印.

    for (int i = 0; i< list.size();i++){
    Object obj = list.get(i);
    System.out.println(obj)
}

增强for循环格式（快捷方式：iter+回车）：底层依赖的是迭代器，增强for其实就是迭代器的简写形式。

​ for（数据类型 变量名：数组或者集合对象）{
​ //循环体，变量即元素
​ }

增强for循环格式遍历List集合

List list = new ArrayList();
list.add(10);
list.add(10);
list.add(20);
for (Object obj:list){
    //obj是集合中的元素，其本身应该是Interger类型的数据
    Integer ii = (Integer) obj;
    System.out.println(ii+10);
}

迭代器

对过程的重复，称为迭代。

迭代器是遍历Collection集合的通用方式，可以在对集合遍历的同时进行添加、删除等操作。

常用方法

  • next():返回迭代的下一个元素对象
  • hasNext() : 如果仍有元素可以迭代，则返回true
List list = new ArrayList();
list.add("a");
list.add("b");
list.add("c");
//1．根据集合对象获取其对象的送代器对象
Iterator it = list.iterator();
//2．判断送代器中是否有元素
while (it.hasNext()){
    //3．如果有就获取元素
    String s = (String)it.next();
    if ("b".equals(s)){ //"b"写前面，这样写可以规避空指针异常
        //如果集合中有b
        //list.add（"java"）；//这样写不行，报并发修改异常，必须调用列表送代器的方法来实现.
    }
    System.out.println(s);
}

注意：

  • 普通的迭代器在遍历集合的同时不能添加或者删除元素，否则会报：并发修改异常
  • 列表迭代器list.listIterator()是List体系独有的遍历方法，可以在对集合遍历的同时进行添加、删除等操作。但是必须通过调用列表迭代器的方法来实现。

泛型

即泛指任意类型，又叫参数化类型（ParameterizedType）
对具体类型的使用起到辅助作用，类似于方法的参数。

表示该集合中存放指定类型的元素

  • 案例演示（给List集合加上泛型String）
    List list = new ArrayList<>()；

  • 泛型的好处：类型安全，避免了类型转换

List<String> list2 = new ArrayList<>();
list2.add("abc");
list2.add("bcd");
for (String s:list2){
    sout(s);
}

总结：

  • 泛型一般只和集合类相结合使用

  • 泛型是JDK5的新特性，但是从JDK7开始，后边的泛型可以不用写具体的数据类型了（菱形泛型

Collections工具类

针对集合进行操作的工具类

成员方法

(大都是静态方法，直接通过Collections.方法调用)

  • sort（List<T>）根据元素的自然顺序，将指定列表按升序排序

  • max（Collection<T>）返回集合的最大元素

  • reverse（List<T>）反转List集合元素

  • shuffle（List<T>）使用默认的随机源随机置换指定的列表，相当于洗牌

    Collections.sort(list);
COllections.max(list);
Collections.reverse(list);
Collections.shuffle(list);

Set集合

  • 特点：不可重复、无序
  • 应用：Set<T> set = new HashSet<>()；
  • 向set中添加元素 add()
  • 遍历集合：迭代器，增强for循环

思考：

Set中加入5个Students对象，然后遍历输出发现并未”去重“？

因为Set集合保证元素的唯一性依赖：equals() 和hashCode()两个方法.

你没有在Student类中重写这两个方法，默认调用的是Object类中的这两个方法.

而Object类中的equals()方法默认比较的是地址值是否相同.

Map集合

特点

  • 双列集合，元素由键值对（Entry）构成：
  • key–value key不可以重复，value可以重复

应用

  • Map<T1，T2> map = new HashMap<>() ；
  • map集合中添加元素：put()
  • 遍历集合的方式：获取所有的key：keySet()，遍历keySet，通过key获取value：get()
  • 遍历keySet的方法：iterator()、增强for
